## はじめに

いつも Microsoft Teams サポートをご利用いただき、誠にありがとうございます。Unified Communications サポート チームです。
Microsoft Teams のご利用中に、想定どおりに動作しない事象が発生した場合、サポートへのお問い合わせは有効な解決方法のひとつです。

そのまま、以下の例のようにご起票いただくことも可能ですが、いくつかのポイント踏まえて整理していただくことでより迅速な疑問・懸念の解消につながります。

> **例)**
> Teams でチャットメッセージが送信できない。

本記事では、弊社サポートへお問い合わせいただく際に状況を整理するポイントとして、以下の 3 点をご紹介します。

- **お問い合わせ内容は具体的に!**
- **背景・経緯も添えて!**
- **再現手順があればご共有を!**

以下より「Teams でチャットメッセージが送信できない」という事象を例に、上記 3 つのポイントをふまえた整理の仕方をご紹介します。 
ぜひ本記事でご紹介するポイントをお問い合わせ起票のご参考にしていただけますと幸いです。

> [!IMPORTANT]
> 本記事でご紹介する情報は、お問い合わせ時にご提供いただくことで調査をスムーズに進めるための参考情報です。
> 事象の内容や調査の進捗によっては、本記事に記載のない追加情報のご提供や、再度の情報採取をお願いする場合がございますので、あらかじめご了承ください。

## 1. お問い合わせ内容は具体的に!

お問い合わせをご起票いただく前に、まずは事象や状況の整理をしましょう。
以下の観点について整理した情報をご提供いただけますと、担当エンジニアがお客様の状況をよりスムーズに把握できるようになります。

- **事象**: 何が起きているのか
- **対象**:
  - 特定ユーザーのみ / 複数ユーザー (xx 人) / 全ユーザー
  - 特定の端末のみ / 複数の端末 (xx 台) / 全端末
- **期間**: いつから事象が発生しているのか
- **頻度**: どの程度の頻度で発生しているのか
- **環境**: 発生しているクライアントやネットワーク

例えば、「Teams でチャットメッセージが送信できない」という事象について、上記の観点で以下のように整理できます。
事象及び環境の詳細については次のセクションもご参考ください。

**<事象・環境情報の例>**

- **事象**:
  - チャットでメッセージ入力後、送信ボタンをクリックしても送信されない
  - 送信中のまま止まり、最終的に「送信できませんでした」と表示される
- **対象**:
  - 特定ユーザーで発生 (example@contoso.com)
  - どの端末を使用しても発生
- **期間**:
  - 12/1 15:34 頃から発生している
- **頻度**:
  - 毎回必ず発生している
- **環境**:
  - Teams デスクトップクライアントのみで発生し、 web クライアントでは発生しない
  - 最新のデスクトップ クライアントでも発生する
  - Windows 11 を使用している端末で発生
  - 社内ネットワークおよび自宅ネットワークで発生している

### 事象について

事象発生時の**画面のスクリーンショットや画面録画をご提供**いただけると、調査の一助となります。
また、画面上にエラーが出ている場合は、**エラー文言やコードをそのままご共有**いただくと、状況を把握しやすくなりますので、お問い合わせ起票時にご提供いただけますと幸いです。

画面録画は Windows の Snipping Tool や、PowerPoint の画面録画機能などで取得可能です。
それぞれの採取方法は以下の公開情報をご参考にしてください。

- [Snipping Tool で画面録画を行う方法](https://support.microsoft.com/ja-jp/windows/snipping-tool-00246869-1843-655f-f220-97299b865f6b)
- [PowerPoint の画面録画機能](https://support.microsoft.com/ja-jp/office/powerpoint-8B-0b4c3f65-534c-4cf1-9c59-402b6e9d79d0)

> [!TIP]
> 以下のような情報も併せてお寄せいただくことで、より迅速に調査を進めるための手掛かりとなりますので、ご提供をご検討ください。
>
> - ユーザー: ユーザーの UPN
> - チーム・チャネル: チーム・チャネルへのリンク
> - 会議/通話: 会議 ID /通話 ID
> - 通話: 事象が発生する発信元、発信先の電話番号

チーム・チャネルへのリンク、会議/通話 ID の採取手順は以下をご参考ください。
<details>
<summary>チーム・チャネルへのリンクの採取方法</summary>
<div style="margin-left:10px;">
1. Teams クライアントを開きます。<br/>
2. 左側より [チーム] アプリを開きます。(ない場合には [チャット] アプリを開きます。)<br/>
3. チームの一覧より、該当チーム・チャネルの右側の [...] より [リンクのコピー] をクリックします。<br/>
4. コピーされたリンクをお寄せください。<br/>
</div>
</details>

<details>
<summary>会議 ID /通話 ID の採取方法</summary>
<div style="margin-left:10px;">
1. Teams 管理センターに管理者権限でサインインします。<br/>
2. 左側の [ユーザー] > [ユーザーを管理] をクリックします。<br/>
3. 会議/通話を行ったユーザーをクリックします。<br/>
4. [会議と通話] タブをクリックします。<br/>
5. "過去の会議" 一覧にございます、事象発生時の会議/通話の会議 ID /通話 ID をお寄せください。<br/>
</div>
</details>

### 環境について

事象が発生する環境については、以下の情報をご提供ください。

- どのクライアントで発生し、どのクライアントでは発生しないのか:
  ※ 切り分けのために複数のクライアントに事象が発生するかご確認ください
  - 例) Teams デスクトップ クライアント、Teams web クライアント、Teams モバイル クライアント
- 最新のバージョンでも発生するのか
  - Teams デスクトップ クライアント
  - Microsoft Edge WebView2
  ※ それぞれのバージョンの確認方法については以下に記載しておりますので、ご参考にしてください。
  <details>
    <summary>Teams デスクトップ クライアントのバージョン確認方法</summary>
    <div style="margin-left:10px;">
    1. Teams デスクトップ クライアントを開きます。<br/>
    2. 最上部の 3 点リーダー [...] より [設定] をクリックします。<br/>
    3. 下部の [Teams] についてをクリックします。<br/>
    4. "バージョン" セクションに表示されております "Microsoft Teams のバージョンは *****.****.****.**** です。" の *****.****.****.**** がデスクトップ クライアントのバージョンです。<br/>
    5. 以下の公開情報に記載されております最新のバージョン以上であるかご確認ください。<br/>
      <div style="margin-left:20px;">
      - Windows: <a href="https://learn.microsoft.com/ja-jp/officeupdates/teams-app-versioning#windows">Version update history for Teams app deployments - Office release notes | Microsoft Learn > Windows</a><br/>
      - Mac: <a href="https://learn.microsoft.com/ja-jp/officeupdates/teams-app-versioning#mac">Version update history for Teams app deployments - Office release notes | Microsoft Learn > Mac</a>
      </div>
    </div>
  </details>
  <details>
    <summary>Microsoft Edge WebView2 のバージョン確認方法</summary>
    <div style="margin-left:10px;">
    1. コマンド プロンプトを開きます。<br/>
    2. 以下のコマンドを入力し、実行します。<br/>
    <div style="padding:10px; background-color:#f4f4f4; border-radius:5px; overflow-x:auto;">
      <div>reg query "HKEY_CURRENT_USER\Software\Microsoft\EdgeUpdate\Clients\{F3017226-FE2A-4295-8BDF-00C3A9A7E4C5}" /v pv</div>
      <div>reg query "HKLM\SOFTWARE\WOW6432Node\Microsoft\EdgeUpdate\Clients\{F3017226-FE2A-4295-8BDF-00C3A9A7E4C5}" /v pv</div>
    </div>
    3. 表示される ***.*.****.** の形式のバージョン番号が Microsoft Edge WebView2 のバージョンです。<br/>
    4. 以下の公開情報に記載されております最新のリリース バージョン以上であるかご確認ください。<br/>
    ※ Microsoft Edge WebView2 は Microsoft Edge のリリース サイクルに合わせて更新されます。<br/>
      <div style="margin-left:20px;">
      - <a href="https://learn.microsoft.com/ja-jp/deployedge/microsoft-edge-release-schedule#microsoft-edge-releases">Microsoft Edge リリース スケジュール | Microsoft Learn > Microsoft Edge リリース</a>
      </div>
    </div>
  </details>
- ご利用端末の OS は何か: 例) Windows 11 / macOS / iOS 18 / Android
- 事象が発生する際に端末が接続しているネットワークの種別は何か: 社内 / 自宅 / キャリア通信

## 2. 背景・経緯も添えて!

お問い合わせの背景や経緯を合わせて記載いただくことで、お客様の課題をより深く理解し、背景に即した回避策や代替手段をご提案しやすくなります。

> **例)**
> [背景・経緯]
> 11/30 頃に Teams デスクトップクライアントを更新してから事象が発生するようになりました。
> 現在チャットを送信する際は、 Teams web クライアントを用いて事象を回避しています。
> ただ可能であればデスクトップクライアントを用いたく、解消策があるか調査して欲しいです。

## 3. 再現手順があればご共有を!

事象が起きるまでの操作が分かる場合は、**できるだけ短く・具体的な再現手順**をご共有ください。
再現手順があると、サポート側で状況の把握や条件の切り分けがしやすくなり、追加で確認すべきポイントを整理する手がかりになります。

> **例)**
> [再現手順]
> 1. Teams デスクトップクライアントを起動します
> 2. 任意のチャットを開きます
> 3. メッセージボックスに「テスト」と入力します
> 4. 送信ボタンをクリックします
> 5. 送信中のまま止まり、「送信できませんでした」と表示されます

## まとめ

おさらいとなりますが、迅速な疑問点、懸念点の解消において特に効果が高いポイントは次の 3 点です。

- **お問い合わせ内容は具体的に!**
- **背景・経緯も添えて!**
- **再現手順があればご共有を!**

分かる範囲の事実からで問題ございませんので、ぜひ本記事でご紹介した上記 3 つのポイントを意識いただけますと幸いです。
この 3 つのポイントを意識することで、「Teams でチャットメッセージが送信できない」という事象も以下のようにまとめることができます。

> **整理前の例)**
> Teams でチャットメッセージが送信できない。

> **整理後の例)**
> [事象]
> チャットでメッセージを入力し、送信ボタンをクリックしても送信されません。
> 送信中のまま止まり、最終的に「送信できませんでした」というエラーメッセージが表示されます。
>
> [対象]
> 特定のユーザーで発生しています。
> 当該ユーザーでは、端末を交換しても発生します。
>
> [期間]
> 12/1 15:34 頃から発生しています。
> 
> [頻度]
> チャット送信時には毎回必ず発生します。
>
> [環境]
> 使用している端末の OS は Windows 11 です。
> Teams デスクトップクライアントで発生しており、 web クライアントでは発生していません。
> また、最新のデスクトップ クライアントでも発生します。
> ネットワークを問わず、社内ネットワークおよび自宅ネットワークで発生しています。
>
> [背景・経緯]
> 11/30 頃に Teams デスクトップクライアントを更新してから事象が発生するようになりました。
> 現在チャットを送信する際は、 Teams web クライアントを用いて事象を回避しています。
> ただ可能であればデスクトップクライアントを用いたく、解消策があるか調査して欲しいです。
>
> [再現手順]
> 1. Teams デスクトップクライアントを起動します
> 2. 任意のチャットを開きます
> 3. メッセージボックスに「テスト」と入力します
> 4. 送信ボタンをクリックします
> 5. 送信中のまま止まり、「送信できませんでした」と表示されます

なお、本記事は上記に準じた記載形式でのお問合せをご起票いただくことを強制するものではございません。
このようにご状況を整理いただくことでユーザーの状況や背景、また今までにご確認いただいた内容をサポート エンジニア側が適切に把握することが可能となり、事象の改善までの時間短縮につながります。
